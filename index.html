chunkFS: A Distributed File System from scratch in Java.
  • Developed a Distributed File System from scratch in Java, featuring a Command Line Interface (CLI) and multithreading Name Server & Chunk Servers. Successfully deployed the system over a local network comprising of Linux and Windows machines.
  • Implemented a Request-Response model for method invocations, similar to RMI technology.
  • Designed a Name Server that relies on a custom database facility (Name Server Registry), storing and managing records in the form of Inode objects.
  • Enforced user management through the “root” user, allowing all users to perform actions like "put" or "get" files, "create" or "remove" directories, and enforcing permissions over their files with "root" user exclusivity.
  • Partitioned files into chunks (up to 5MB) for uploading and sent them serially to the chunk servers via the name server upon initiation of “put” request. Recollected those chunks from the chunk servers via the name server upon a “get” request and correctly reconstructed the file to save it.
  • Distributed and replicated chunks by following Round-Robin strategy.
  • Assigned canonical names to the chunk servers to maintain consistency in the name server's registry even after the chunk server has been relocated to another IP address.
  • Assigned dual roles to the chunk servers to behave as primary and/or secondary storage servers.
  • Configurable aspects (IP address, port, etc.) of the Name Server, Chunk Server, and the CLI can be modified through dedicated configuration files.

Raspberry Pi 3B+ bootable image: GPIO and Mini-UART drivers
Raspberry Pi 3B+ bootable image: GPIO and Mini-UART driversRaspberry Pi 3B+ bootable image: GPIO and Mini-UART drivers In my quest to learn RTOS development using C, I attempted to develop a small bootable image for the Raspberry Pi 3B+ based on Broadcom’s BCM2837B0 SoC and ARM’s Cortex-A53 quad-core processor. Key accomplishments include:
  • development of a boot loader script in assembly language following the ARMv8-A instruction set for lowering down the exception level of the main core from EL3 to EL1.
  • booting of the image at the physical address 0x0 as opposed to the default address 0x80000 through a custom linker script.
  • making all the secondary cores spin in a loop, delegating all the tasks thereafter to the main core.
  • assigning a 512-byte stack to the main core and making the main core jump from the assembly code to the “void cpu0_main(void)” function in C.
  • development of a minimal mini-UART driver for the display of messages onto the console.
  • development of a minimal GPIO driver for controlling hardware components.
  • development of an imitated version of “stdint.h” and “stdbool.h” headers for handling register addresses and program-wide constants.
  • development of “delay.h” header for software-enforced delay, synchronizing its working with processor’s operating frequency
  • development of a batch script for automatic build of the image in a single click.
The bootable image was emulated on QEMU, and the registers were thoroughly observed by interfacing QEMU with the GDB debugger over the localhost via port 1234.

References for developing the drivers were drawn from the Broadcom BCM2837B0 peripheral manual, and the “aarch64-none-elf” toolchain (cross-compiler) was employed for building the image.

MiniURL: An URL shortener web application in PHP using MVC architecture.
I developed a PHP-based web application leveraging the MVC architecture with a custom router implementation that enables users to shorten their URLs. The application was deployed over a local network (192.168.43.0/24), comprised of computers with distinct roles:
  • 192.168.43.45/24: MySQL Database Server
  • 192.168.43.67/24: Apache Web Server
  • 192.168.43.10/24: DNS Server
  • Other IPs in the network 192.168.43.0/24 (except broadcast and network): Client
Assigned domain names “www.miniurl.com” to 192.168.43.67/24 (the Apache Web Server) and “database.miniurl.com” to 192.168.43.45/24 (the MySQL Database Server) via the BIND9 DNS server on Ubuntu 20.04 LTS. This application:
  • allows anonymous users to shorten their URLs, view the click-count of their links, and create accounts.
  • offers extra control over the shortened URLs to registered users: modifying aliases, temporary deactivation, and permanent deletion of the URL(s).
  • generates shortened URLs like "www.miniurl.com/u/urlid", with a 5-character long "urlid" (customizable with a length 1-10)
  • allows registered users to change their account passwords and delete their accounts.
  • utilizes PHP’s built-in $_SESSION variable for effective context management across web pages.
  • ensures persistent login with PHP's $_COOKIE variable even after the browser has been closed.

ML++: A vectorized machine learning library for regression and classification
This serves as my final semester project and is currently in progress. I have chosen C++ for this project as it supports "operator overloading", which is useful for software-level "vectorization" and is very much coupled to the hardware, which accounts for speed. Moreover, compilers for C++ exist for almost any target architecture, and the programs can be optimized for performance and instructions.
    This library will:
  • imitate "vectorization" through operator overloading, accounting for fast computations.
  • be able to perform regression and/or classification (as applicable) over multi-variable and multi-variate datasets.
  • consist of a CSV parser/reader for representing datasets.
  • consist of a DataFrame object for manipulating the dataset: dropping of columns, partitioning of the dataset, normalization & standardization of input features, and one-hot encoding of the output.
  • plot an interactive graph using "matplotlibcpp.h" for the loss V/s epoch and compute metrics (MSE or RMSE) for assessing the model's performance.
